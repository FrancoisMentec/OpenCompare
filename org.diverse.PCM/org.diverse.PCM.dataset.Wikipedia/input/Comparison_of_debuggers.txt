==List of debugger front-ends==

* Many [[Eclipse (software)|Eclipse]] perspectives, e.g. the Java Development Tools (JDT) [http://www.eclipse.org/jdt/index.php], provide a debugger front-end.
* [[Allinea Distributed Debugging Tool|Allinea DDT]] - a graphical debugger supporting for parallel/multi-process and multithreaded applications, for C++/C++ and F90.
* [[Data Display Debugger|DDD]] is the standard front-end from the [[GNU Project]]. It is a complex tool that works with most common debuggers (GDB, jdb, Python debugger, Perl debugger, Tcl, and others) natively or with some external programs (for PHP).
* [[GDB]] (the GNU debugger) GUI
** [[Affinic Debugger GUI]] &mdash; A GDB Frontend on Linux/Windows/OSX/Cygwin.
** [[Emacs]] &mdash; Emacs editor with built-in support for the GNU Debugger acts as the frontend.
** [[KDbg]] &mdash; Part of the KDE development tools.
** [[Nemiver]] &mdash; A GDB frontend that integrates well in the [[GNOME]] desktop environment.
** [[xxgdb]] &mdash; X-window frontend for GDB and [[dbx debugger]].
** [[Qt Creator#Debuggers|Qt Creator]] &mdash; multi-platform frontend for GDB, CDB and LLDB ([http://qt-project.org/doc/qtcreator-2.8/creator-debug-mode.html]).
** [http://cgdb.sourceforge.net/ cgdb] &mdash; ncurses terminal program that mimics [[Vim (text editor)|vim]] key mapping.
** [http://ccdebug.sourceforge.net/ ccdebug]&mdash; A graphical GDB frontend using the [[Qt toolkit]].
** [[Padb]] &mdash; has a parallel front-end to GDB allowing it to target parallel applications.
** [[Allinea Distributed Debugging Tool|Allinea's DDT]] &mdash; a parallel and distributed front-end to a modified version of GDB.
** [[Xcode]] &mdash; contains a GDB front-end as well.
** [[SlickEdit]] &mdash; contains a GDB front-end as well.
** [[Eclipse (software)|Eclipse]] C/C++ Development Tools (CDT) [http://www.eclipse.org/cdt/] &mdash;  includes visual debugging tools based on GDB.
** [[CodeLite]] &mdash; An open source, cross platform C/C++ IDE which have front end for [[gdb]], the next version of [[CodeLite]] (v6.0) will also include a front end to the [[LLDB (debugger)]]
